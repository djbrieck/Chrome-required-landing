# Google Chrome Required Landing Page  

A tiny static website that announces “Google Chrome is Required” and links to major streaming/learning platforms (Netflix, Spotify, Disney Plus, Udemy). The page is stored on IPFS via the `deploy.sh` script.  

---  

## Project Overview  

| Item | Description |
|------|-------------|
| **Purpose** | Provide a quick landing‑page experience that reminds visitors Chrome must be installed to access popular services. |
| **Target audience** | General users who may not have Chrome; the page is meant for demonstration or troubleshooting. |
| **Storage backend** | Files are added to an IPFS cluster using `ipfs-cluster-ctl`. A public hash (`PUBLIC_HASH`) is generated and printed by the script. |
| **Client file** | `index.html` – a single‑page HTML page with minimal styling. |

## How the Deployment Works  

1. **Run the script** – `./deploy.sh`  
2. The script validates that `index.html` exists.  
3. It calls `ipfs-cluster-ctl add` to push the file into the cluster:  

```bash
PUBLIC_HASH=$(ipfs-cluster-ctl add "index.html" \
    --name "Google Chrome Required Landing Page" \
    --metadata type=website -Q)
```

4. The script prints the resulting public hash, e.g.:

```
Successfully added selected files.
Received public hash of: QmX... (replace with actual hash)
```

5. After deployment you can view the page at:  

`https://gateway.ipfs.io/ipfs/&lt;PUBLIC_HASH&gt;`  
(Replace `&lt;PUBLIC_HASH&gt;` with the value printed by the script.)

## Prerequisites  

- **Node.js / Bash** – to run `deploy.sh`.  
- **`ipfs-cluster-ctl`** binary installed and accessible in `$PATH`.  
  (Usually part of the IPFS cluster manager you are already using.)  

If any of these tools are missing, install them first.

## FAQ / Gotchas  

| Question | Answer |
|----------|--------|
| **Why does it say “Chrome Required”?** | The page intentionally informs users they need Chrome to open services such as Netflix, Spotify, Disney Plus, Udemy. It’s a demo of a minimal HTML site. |
| **What if I don’t have IPFS running?** | `ipfs-cluster-ctl` is just a wrapper around the main `ipfs add` command. You’ll need an active IPFS cluster or at least the `add` RPC enabled. |
| **Can I view the page elsewhere?** | The URL will always point to the hash generated by your specific `deploy.sh` run (it may change if you re‑run). |
| **Why only one file?** | Keeping it simple: a single HTML file is enough for this demo; any additional assets would require updating the script. |
